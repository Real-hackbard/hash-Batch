A file hash is a unique digital fingerprint of a file, generated by a mathematical hash function. Regardless of the file size, this code always has a fixed length. Even the slightest change to the file (such as a single space or bit) results in a completely different hash value.

<img src="https://github.com/user-attachments/assets/13af1bdf-3b4c-4084-acff-2e47f3bc23b2" />

</br>

### Main areas of application
* Integrity check: Verify whether a file has been corrupted or tampered with during download or transfer.
* Unique identification: Detecting duplicates in systems and databases, since identical files always generate the same hash.
* Security scans: Antivirus programs compare file hashes against databases of known malware.
* Version control: Systems like [Git](https://wiki.archlinux.org/title/SHA_hashes) use hashes to precisely track code changes and commits.

# Hash algorithms
DCPcrypt is a collection of cryptographic components for the Borland Delphi(tm), C++ Builder(tm) and Kylix(tm) programming languages. The supported versions are Delphi 4, 5, 6 and 7, C++ Builder (3?), 4, 5, 6 and Kylix 1 (untested) and 2.

The idea behind DCPcrypt is that it should be possible to "drop in" any algorithm implementation to replace another with minimum or no code changes. To aid in this goal all cryptographic components are descended from one of several base classes, TDCP_cipher for encryption algorithms and TDCP_hash for message digest algorithms.

DCPcrypt is open source software (released under the [MIT license](https://is.muni.cz/th/51468/fi_b/priloha/DCPcrypt/Docs/MIT_license.txt)) and as such there is no charge for inclusion in other software.
